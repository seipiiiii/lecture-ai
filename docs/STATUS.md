## 解決済み課題

### BUG-001: レイアウトのズレ（GitHub Pages）
- **症状**: デスクトップ表示でヘッダー・タブバーが左寄りになる
- **原因**: `position: fixed; left: 0; right: 0; margin: 0 auto` は中央寄せにならない
- **解決**: `left: 50%; transform: translateX(-50%)` に変更
- **関連**: `@keyframes fadeIn` の `transform` が `translateX(-50%)` を上書きしていたため、アニメーションを `opacity` のみに変更

### BUG-002: `catch {}` 構文エラー
- **症状**: 一部ブラウザで JS が全て動作しない
- **原因**: ES2019 の optional catch binding `catch {}` が古いブラウザ非対応
- **解決**: `catch (e) {}` に統一

### BUG-003: 「分析する」ボタンが押せない（ファイル選択後）
- **症状**: 音声ファイル選択後もボタンが反応しない
- **根本原因**: `<input type="file" hidden>` の `hidden` 属性により、モバイルブラウザで `fileInput.click()` がブロックされファイルが選択されない
- **解決**: `hidden` 属性を除去し、CSS（`position:absolute; width:1px; height:1px; opacity:0`）で非表示化

### BUG-004: ボタンクリックが無反応（モバイル本番環境）
- **症状**: ファイル選択後にボタンが緑（有効）に見えるが、タップしても反応なし
- **根本原因**: HTML の `disabled` 属性付きボタンはブラウザがネイティブレベルでクリックイベントをブロックする。何らかの理由で `disabled` が外れていなかった
- **解決**: `disabled` 属性を完全廃止し、CSS クラス `btn-inactive` で管理。クリックは常に発火し、JS 側でクラス判定

### BUG-005: APIキー検証で 429 → 「無効」と誤判定
- **症状**: レート制限中にAPIキーを入力すると「無効なキー」エラーが出てキーが保存されない
- **解決**: 検証レスポンスで `res.status === 429` を有効として扱うよう変更

### BUG-006: 404 エラー（モデル未存在）
- **症状**: 分析時に HTTP 404 が返る
- **原因**: `gemini-2.5-flash-preview-04-17` は2025年4月のプレビュー版。2026年時点で廃止済み
- **解決**: モデルを `gemini-2.5-flash`（正式版）に変更

### BUG-007: モデル互換性（APIキー検証）
- **症状**: スマホでAPIキー入力時に「APIキーが無効」エラー
- **原因**: 検証リクエストに `gemini-2.5-flash` を使用しており、プレビュー中のキーでは利用不可の場合がある
- **解決**: 検証は安定版 `gemini-2.0-flash` を使用し、分析は `gemini-2.5-flash` を維持

## 未解決課題・懸念事項

### WARN-001: 429 レート制限（本番環境）
- **状況**: 無料プランでは RPM 制限が低く、分析時に 429 が頻発
- **影響**: ユーザーが分析できない場合がある
- **対策案**:
  - ユーザーに課金設定を促す
  - エクスポネンシャルバックオフでリトライ実装
  - ファイルサイズ上限を下げて処理トークンを削減

### WARN-002: ブラウザキャッシュによる旧バージョン混在
- **状況**: GitHub Pages CDN キャッシュにより HTML と JS のバージョン不整合が発生
- **対策**: スクリプトタグに `?v=N` クエリを付加（現在 v=11）。変更時はバージョンを上げること

### WARN-003: Base64 inline_data の制限
- **状況**: 大きな音声ファイルはリクエストボディが肥大化し API エラーになる可能性
- **対策案**: Gemini File API（別途アップロード→参照）への移行を検討

## 詰まりの記録

### 2026-04: ボタン無反応の調査
最も時間がかかった問題。プレビュー環境では動作するが本番（GitHub Pages）では動作しないという再現困難なバグ。

調査の流れ:
1. ファイル選択が動作しない → `hidden` 属性が原因 → CSS 非表示化で解決
2. ボタンタップ無反応 → JS イベントリスナーは登録済み → `disabled` 属性のブラウザネイティブブロックが原因
3. エラーが見えない → トーストが消える前に見逃していた → UI 内固定エラー表示に変更
4. キャッシュによる HTML/JS 不整合 → スクリプトにバージョンクエリ付加で対応
