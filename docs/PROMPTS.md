# PROMPTS — AIへの重要指示・プロンプト設計

## Gemini API へのプロンプト

### 1. 文字起こしプロンプト

```
この音声を日本語で文字起こししてください。全文をそのまま出力してください。
音声が聞き取れない部分は「（聞き取れず）」と記載してください。
```

**設計根拠**:
- 「全文をそのまま出力」で要約・省略を防ぐ
- 聞き取れない部分の扱いを明示してハルシネーションを抑制
- 音声は `inline_data` で送信（Base64 エンコード）

---

### 2. 要約・要点抽出プロンプト

```
以下の講義の文字起こしを分析し、以下の形式でJSONのみを出力してください。
説明文や```は不要です。
{
  "summary": "3行以内の要約（改行で区切る）",
  "keyPoints": ["要点1", "要点2", "要点3"]
}
【文字起こし】
{transcriptionText}
```

**設計根拠**:
- **「JSONのみを出力」「説明文や\`\`\`は不要」**: Gemini はマークダウンのコードブロックで JSON を返すことが多い。パース前に `replace(/```json|```/g, '').trim()` で除去するが、プロンプトレベルでも抑制
- **JSON 形式の明示**: 構造化出力のため
- **パース失敗時のフォールバック**: 正規表現 `/\{[\s\S]*\}/` でオブジェクト抽出
- **`temperature: 0.4`**: 低めの温度で安定した要約を生成

**参考**:
- https://ai.google.dev/gemini-api/docs/structured-output

---

### 3. クイズ生成プロンプト

```
以下は大学講義の文字起こしです。学習者向けの予想問題を5問作成してください。
【重要】JSON配列のみを返してください。説明文や```は不要です。
[
  {
    "question": "問題文",
    "options": ["A: 選択肢", "B: 選択肢", "C: 選択肢", "D: 選択肢"],
    "answer": "A",
    "explanation": "解説文（2〜3文）"
  }
]
【文字起こし】
{transcriptionText}
```

**設計根拠**:
- **「JSON配列のみ」**: オブジェクトではなく配列を要求し、ループ処理を簡単に
- **「予想問題」**: 試験対策に使えることを示唆
- **選択肢形式 `"A: 選択肢"`**: 正解キー（"A"）と選択肢テキストを分離管理しやすい形式
- **解説必須**: 学習効果のため
- **`temperature: 0.7`**: 多様な問題生成のためやや高め
- **パース失敗時のフォールバック**: 正規表現 `/\[[\s\S]*\]/` で配列抽出

---

### 4. APIキー検証プロンプト

```
{ "contents": [{ "parts": [{ "text": "hi" }] }] }
```

**設計根拠**:
- 最小限のリクエストでキーの有効性を確認
- モデル: `gemini-2.0-flash`（安定版。2.5 Flash は廃止リスクがあるため検証には不使用）
- HTTP 429（レート制限）は「キー有効」として扱う

## Claude Code への重要指示（開発時）

### CSS 中央寄せ

```
position: fixed の要素を中央寄せする場合は
left: 0; right: 0; margin: auto ではなく
left: 50%; transform: translateX(-50%) を使うこと。
GitHub Pages のデスクトップ表示でズレが発生するため。
```

**根拠**: `margin: auto` は `position: fixed` では機能しない（通常フローの要素のみ有効）

### アニメーション

```
fadeIn アニメーションには transform を使わないこと。
translateX(-50%) と競合してレイアウトが崩れる。
opacity のみでフェードインを実装すること。
```

### モバイル file input

```
<input type="file"> に hidden 属性を付けてはいけない。
モバイルブラウザで fileInput.click() がブロックされる。
CSS で非表示にすること:
style="position:absolute;width:1px;height:1px;opacity:0;overflow:hidden;"
```

**根拠**: iOS Safari の制限。`hidden` 属性はブラウザがネイティブレベルでインタラクションをブロック

### ボタン disabled 属性

```
分析ボタンの有効/無効管理に HTML の disabled 属性を使わないこと。
disabled ボタンはブラウザがネイティブでクリックイベントをブロックする。
CSS クラス（btn-inactive）で管理し、クリックは常に発火させること。
```

### ES バージョン

```
optional catch binding (catch {}) は使わないこと。
古いブラウザで構文エラーになり JS 全体が動作しなくなる。
必ず catch (e) {} と変数を付けること。
```

### キャッシュバスティング

```
JS ファイルを変更した際は index.html のスクリプトタグの
?v=N クエリパラメータを上げること。
GitHub Pages CDN と ブラウザキャッシュの不整合を防ぐため。
```
