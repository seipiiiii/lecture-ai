## TC-001: 初回起動・APIキー設定

**前提**: localStorage が空（シークレットモードで開く）

| ステップ | 操作 | 期待結果 |
|---|---|---|
| 1 | GitHub Pages URL を開く | セットアップ画面が表示される（ヘッダー・タブバーは非表示） |
| 2 | APIキー入力欄に無効なキーを入力して「始める」 | 「APIキーが無効です」エラー表示 |
| 3 | 有効な Gemini APIキーを入力して「始める」 | ローディング表示後、メイン画面へ遷移。トースト「APIキーを設定しました」 |
| 4 | 設定アイコン（右上）をタップ | 設定モーダルにAPIキーが表示されている |

**証跡**: screenshots/tc001_setup.png

## TC-002: 音声ファイル選択

**前提**: APIキー設定済み、ホーム画面表示中

| ステップ | 操作 | 期待結果 |
|---|---|---|
| 1 | ドロップゾーンをタップ | ファイルピッカーが開く |
| 2 | .m4a ファイルを選択 | ドロップゾーンにファイル名・サイズが表示される |
| 3 | 「分析する」ボタンの状態確認 | ボタンが緑色（不透明）になりタップ可能 |
| 4 | × ボタンをタップ | ファイルが解除され、ドロップゾーンが初期状態に戻る |

**対応形式チェック**:
- [x] .mp3
- [x] .wav
- [x] .m4a
- [x] .mp4
- [x] .webm
- [x] .ogg
- [ ] .txt（非対応 → エラートースト）
- [ ] 21MB以上（非対応 → エラートースト）

## TC-003: 講義分析（正常系）

**前提**: APIキー設定済み、30秒〜2分程度の音声ファイル選択済み

| 4 | クイズ処理中 | 「クイズを生成中…」テキストとプログレス 80% |

## TC-004: 講義分析（エラー系）

| ケース | 条件 | 期待結果 |
|---|---|---|
| 429 エラー | レート制限中に分析実行 | 赤いエラーボックス「⚠️ 通信エラーが発生しました（HTTP 429）」。1〜2分後に再試行可能 |
| APIキーなし | APIキー未設定で分析ボタンタップ | トースト「APIキーを設定してください」→ 設定モーダルが開く |
| ファイルなし | ファイル未選択で分析ボタンタップ | ボタンが非アクティブ（不透明度40%）なため反応なし |
